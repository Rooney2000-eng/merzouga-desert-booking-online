# دليل نشر موقع Sahra Joy أونلاين بدون سيرفر خاص

هذا المشروع جاهز لكي يعمل على **Vercel** بدون VPS وبدون سيرفر تقليدي.

## أسرع طريقة للنشر

### 1) ارفع المشروع إلى GitHub
ارفع **كل الملفات والمجلدات** داخل المشروع، خصوصا:

- `app`
- `components`
- `lib`
- `public`
- `package.json`

### 2) ادخل إلى Vercel
سجّل الدخول بحساب GitHub.

### 3) استورد المشروع
اختر Repository الخاص بك ثم اضغط **Deploy**.

### 4) بعد النشر
ستحصل على رابط مثل:

```text
https://your-project.vercel.app
```

## الدومين الاحترافي sahrajoy.com

بعد نجاح النشر:

1. افتح إعدادات المشروع في Vercel
2. ادخل إلى **Settings > Domains**
3. أضف:
   - `sahrajoy.com`
   - `www.sahrajoy.com` (اختياري)
4. نفذ سجلات DNS المطلوبة عند شركة الدومين
5. بعد التفعيل، سيكون الموقع متاحا على دومينك الحقيقي

## هل أحتاج قاعدة بيانات الآن؟

لا.

النسخة الحالية تعمل حتى بدون:
- Supabase
- Resend
- دفع إلكتروني

لأن الحجز حاليا يتم هكذا:

1. الزائر يملأ النموذج
2. يظهر له زر واتساب برسالة جاهزة
3. يتم تأكيد الحجز معك مباشرة عبر واتساب

## متى أحتاج Supabase أو البريد؟

تحتاجها فقط إذا أردت لاحقا:

- حفظ كل الطلبات في قاعدة بيانات
- استقبال إشعار بريدي تلقائي
- إدارة الحجوزات داخل لوحة تحكم

## متغيرات البيئة الحالية

القيم الأساسية مضبوطة داخل الموقع بالفعل:

- `NEXT_PUBLIC_SITE_URL=https://sahrajoy.com`
- `NEXT_PUBLIC_BRAND_EN=Sahra Joy`
- `NEXT_PUBLIC_BRAND_AR=صحرا جوي | رحلات مرزوكة`
- `NEXT_PUBLIC_WHATSAPP_NUMBER=212691999897`
- `NEXT_PUBLIC_CONTACT_EMAIL=Soufianechahid30@gmail.com`

لذلك يمكنك النشر الآن حتى بدون إعداد `.env.local`.

## إذا أردت تعديل القيم لاحقا

انسخ `.env.example` إلى `.env.local` وعدّل ما تريد.

## ملاحظات عملية

- إذا ظهر خطأ في Vercel، تأكد أن جميع المجلدات رُفعت إلى GitHub
- لا ترفع ملف ZIP نفسه إلى GitHub، بل ارفع الملفات التي بداخله
- إذا عدلت الملفات في GitHub ثم كان الموقع مفتوحا على Vercel، قم بعمل **Redeploy**
